# Tier Trophy Cascade Plan

## Goal
Implement tier cascading so higher-tier subscription holders automatically unlock lower-tier trophies. This increases perceived value for founding trainers and lifetime members.

## Cascade Rules

| User Tier | Unlocked Trophies |
|---|---|
| `founding_alpha` | founding_alpha, founding_beta, founding_gamma, champion, gym_leader |
| `founding_beta` | founding_beta, founding_gamma, champion, gym_leader |
| `founding_gamma` | founding_gamma, champion, gym_leader |
| `lifetime` | champion, gym_leader |
| `annual` | gym_leader |

## Files to Modify

### `lib/trophies.ts`
- Add `TIER_CASCADE_MAP: Record<string, string[]>` mapping each subscription tier to the trophy IDs it unlocks
- This is a pure data addition, no logic changes needed here

### `app/api/user/trophies/route.ts` (GET handler)
- Replace the simple `trophy.tier === userTier` check with a `TIER_CASCADE_MAP` lookup
- For each tier trophy, check if `userTier` is in the cascade map that includes `trophy.tier`
- Use subscription start date as `unlockedAt` for all cascading unlocks

### `app/api/user/trophies/route.ts` (POST handler)
- No changes needed — tier trophies are never POST-unlocked via the API (they derive from subscription state)

## Implementation Details

### GET handler logic change

Current:
```ts
const isTierUnlocked = trophy.category === "tier" && trophy.tier === userTier
```

New:
```ts
const cascadedTiers = TIER_CASCADE_MAP[userTier] || []
const isTierUnlocked = trophy.category === "tier" && cascadedTiers.includes(trophy.tier!)
```

### Tier cascade map (in lib/trophies.ts)
```ts
export const TIER_CASCADE_MAP: Record<string, string[]> = {
  founding_alpha: ["founding_alpha", "founding_beta", "founding_gamma", "lifetime", "annual"],
  founding_beta:  ["founding_beta", "founding_gamma", "lifetime", "annual"],
  founding_gamma: ["founding_gamma", "lifetime", "annual"],
  lifetime:       ["lifetime", "annual"],
  annual:         ["annual"],
}
```

Note: the `tier` field on `TrophyDefinition` maps to subscription tier IDs (`founding_alpha`, `founding_beta`, `founding_gamma`, `lifetime`, `annual`), not the trophy IDs. The `TIER_TROPHIES` array has:
- `founding_alpha` → tier: `founding_alpha`
- `founding_beta` → tier: `founding_beta`
- `founding_gamma` → tier: `founding_gamma`
- `champion` → tier: `lifetime`
- `gym_leader` → tier: `annual`

So the cascade map keys are subscription tiers, and the values are the `trophy.tier` strings to match against.

## Testing
- GET /api/user/trophies for a `founding_alpha` user should return all 5 tier trophies as unlocked
- GET /api/user/trophies for a `lifetime` user should return champion + gym_leader as unlocked
- GET /api/user/trophies for a `free` user should return 0 tier trophies unlocked
- Verify `unlockedAt` is set to subscription start date for all cascaded unlocks
